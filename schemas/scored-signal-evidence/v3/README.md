# Canonical Scored-Signal Evidence Contract (v3)

**Family:** `afi.scored-signal-evidence.v3` — the canonical scored-signal evidence record, **with provider invocation provenance**.
**Status:** `governed-contract` — the **active write contract and the SOLE current evidence contract** of the one canonical scored-signal evidence store.
**Authorization:** `afi-governance/decisions/evidence-v3-provider-provenance-v0.1` (EV3-GOV), exercising the contract family's own change-control rule under FCP-GOV D-FCP-3 ("any shape change requires a new governance decision and a new schema version — never a silent mutation").

## Sole-current status (EV3-GOV D-EV3-1 / D-EV3-8)

- **v3 is the only current evidence contract**: every newly written canonical evidence record conforms to v3, and the canonical store admits **only** v3 (D-EV3-7, recomputation-verified).
- There is **no dual mode, no feature-flagged fallback, no compatibility alias, and no back-write**.
- The prior v1/v2 versions ceased to be active contracts, and their active-tree surfaces (schema directories, examples, vectors, tests) were **deleted forward-only** under D-EV3-8 — no archive directories, no deprecation markers, no tombstones. **Git history and the accepted governance ledger are the archive.**

## Delta over the carried-forward v2 shape

v3 is the v2 record shape **carried forward unchanged** — the same reused `afi.scored-signal.v1` / `afi.provenance-record.v1` / `CanonicalHash` / [`afi.composition-ref.v1`](../../composition-ref/v1/composition-ref.schema.json) `$ref`s, the same twelve required and two optional properties, the same lifecycle/finality `if/then/else` binder, the same store-layer and continuity constraints — **plus exactly three REQUIRED additions**:

| Delta | Value |
|---|---|
| `schema` const | `afi.scored-signal-evidence.v3` |
| **new REQUIRED** `providerInvocations` | positional five-tuple of [`afi.provider-invocation-proof.v1`](../../provider-invocation-proof/v1/provider-invocation-proof.schema.json) — order `aiMl, news, pattern, sentiment, technical` (D-EV3-2) |
| **new REQUIRED** `recordHash` | `CanonicalHash v1`, domain `afi.d2.evidence-record` (D-EV3-4) |
| **new REQUIRED** `replayHash` | `CanonicalHash v1`, domain `afi.d2.evidence-replay` (D-EV3-4) |

## What the additions add

- **`providerInvocations`** — exactly five closed, credential-safe proofs of the invocations that occurred, one per governed category, unique and deterministically ordered (structurally enforced by the positional tuple). Each binds provider/instance/adapter/credential identity, the normalized invocation-input hash, the provider-result hash, and the category-result hash that entered the join; the `aiMl` proof additionally nests [`afi.aiml-invocation-proof.v1`](../../aiml-invocation-proof/v1/aiml-invocation-proof.schema.json) (D-EV3-3). Proof facts are **carried, never consumed**.
- **`recordHash`** — the full-record integrity commitment: the record minus `{recordHash, replayHash}` under `canonical-json-hashing.v1` (tag carried, never hashed). Verified by recomputation before insert.
- **`replayHash`** — the deterministic semantic/replay commitment: the record minus `{recordHash, replayHash, lifecycleState, finalized, recordVersion, supersedesRecordHash}`. Identical canonical inputs through the same composition yield identical values; lifecycle progression never moves it.

Everything else — boundary (schema/contract only), store-layer constraints, identifier continuity, the analyst-neutral UWR profile stamp (PR-UWR-STAMP shape, RC-6 `source` discriminator), the all-or-nothing composition reference — is **carried forward unchanged**.
